## Key Insights & Lessons Learned 💡

### Mathematical Modeling
1. **Isomorphic Calculations:** Sharing math logic between client/server prevents bugs and drift
2. **Pure Functions:** Stateless calculation functions are easy to test and reason about
3. **Explicit Rounding:** Always round explicitly rather than relying on display formatting

### Data Integrity
1. **Bottle Registry as Source of Truth:** Centralizing geometry data prevented inconsistencies
2. **USDA Standards:** Using official USDA data ensured nutritional accuracy
3. **Type Safety:** TypeScript interfaces caught data mismatches at compile time

### User Experience
1. **Multiple Units:** Providing ml, tbsp, and cups accommodates different user preferences
2. **Decimal Precision:** 1 decimal place for tbsp/cups, 0 decimals for ml balances precision and readability
3. **Consumed vs. Remaining:** Showing both values helps users understand usage patterns
