## Key Insights & Lessons Learned 💡

### Data Pipeline Design
1. **Validation Early:** Layer 1 validation at submission time prevents bad data from entering pipeline
2. **Confidence Weighting:** Graduated confidence weights (not binary accept/reject) preserve more data
3. **Training Eligibility:** Automatic flagging reduces manual data curation effort

### Storage Strategy
1. **Hybrid Approach:** Structured data in Supabase, blobs in R2 optimizes cost and query performance
2. **Blurhash Privacy:** Image hashing provides privacy benefit while enabling visual verification
3. **Metadata Co-location:** Storing scan metadata with feedback simplifies training data preparation

### User Feedback UX
1. **1-Tap Happy Path:** Most users select "About right" - optimize for this case
2. **Optional Correction:** Don't force slider adjustment - accuracy rating alone is valuable
3. **Fast Submission:** Sub-1s feedback submission prevents user drop-off
