## Key Insights & Lessons Learned 💡

### Multi-Item Management
1. **State Complexity:** Managing multiple items requires careful state architecture
2. **Unique Identifiers:** Each bottle needs stable unique ID for tracking
3. **User Limits:** Reasonable limits (5-10 bottles) prevent UI/UX issues

### Component Architecture
1. **Manager Pattern:** Bottle manager component encapsulates multi-item logic well
2. **State Persistence:** IndexedDB persistence essential for multi-bottle tracking
3. **Component Reuse:** Existing single-bottle components adapted easily

### User Value
1. **Real-World Use Case:** Users often have multiple bottles (different oils, sizes)
2. **Convenience:** Multi-bottle tracking eliminates need to re-scan QR codes
3. **Comparison:** Users compare consumption across bottles
