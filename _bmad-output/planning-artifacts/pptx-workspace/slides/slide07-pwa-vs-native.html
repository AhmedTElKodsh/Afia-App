PWA vs Native App

OUR CHOICE ✓

Progressive Web App (PWA)

Used for: POC v1 + Full Launch

✓

No App Store submission

QR code → instant access in browser · No install required

✓

Single codebase for iOS + Android

React + Vite · Deployed as static files on Cloudflare Pages

✓

Camera API available in browser

getUserMedia() · Works on Chrome Android + Safari iOS

✓

Offline shell via Service Worker

App loads instantly from cache · Only scan needs network

!

iOS caveat: Use Safari browser mode

Standalone PWA mode has a known iOS camera bug → "Open in Safari" prompt shown

PHASE 4 OPTION

Native Mobile App

React Native or Flutter · After 1000+ scans

✗

App Store review required

Apple + Google review process · Days to weeks to publish

✗

Separate iOS + Android codebases

Or React Native (adds complexity vs current stack)

✓

Full camera + hardware access

No iOS PWA limitation · Better UX for power users

✓

Push notifications

Reminders to track oil · Engagement features (Phase 4)

✗

Higher dev cost + time

Not justified until product-market fit is proven

Decision: PWA for POC + Full Launch. Native app considered only after 1,000+ real scans validate product-market fit.
